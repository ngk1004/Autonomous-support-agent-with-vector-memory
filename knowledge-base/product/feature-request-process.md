# Feature Request Process

We welcome feature requests from all customers. Here's how your ideas reach our product team.

## Submit a Feature Request

1. **In-app**: Click the **?** help menu → **Request a Feature**
2. **Portal**: https://feedback.acmecloud.example
3. **Email**: product@acmecloud.example

Include:

- Problem you're trying to solve
- Proposed solution (optional)
- How critical this is to your workflow
- Your plan tier (helps us prioritize enterprise needs)

## What Happens Next

| Stage | Timeline | Description |
|-------|----------|-------------|
| Triage | 1–3 days | Product team reviews and categorizes |
| Under Consideration | Ongoing | Added to backlog if aligned with strategy |
| Planned | Quarterly | Selected for an upcoming release cycle |
| In Progress | Varies | Engineering is building it |
| Shipped | — | Released; you get notified if you voted |

## Voting

Upvote existing requests on the feedback portal. Popular requests influence prioritization but do not guarantee implementation.

## Enterprise Feature Requests

Enterprise customers with dedicated CSMs can escalate via their CSM for quarterly roadmap reviews.

## What We Typically Don't Build

- Custom one-off integrations (use our API or partner marketplace)
- Features outside our core platform scope
- Requests without clear use cases from multiple customers

## Slack Integration Request

Slack integration is one of our most requested features. It is currently **Under Consideration** for Q3. Vote at feedback.acmecloud.example/slack to increase priority.
